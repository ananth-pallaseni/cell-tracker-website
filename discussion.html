Discussion

Segmentation

The segmentation pipeline, as it stands, can successfully segment HSCs in image stacks that are not too noisy. The system is lightweight and speedy but it does, however, suffer from two major flaws: a large number of false positives, and a reliance on some manual tuning of parameters.

False Positives: While the cells are succesfully detected, they are often accompanied by assorted detritus that the system picks up as viable cell candidates. It is quite apparent to the human eye that these are not cells, however the differentiating factor is not any straightforward measurement that I could find. I suspect that more work on integrating the time-varying aspect of the data could go some way to fixing this.

Manual Tuning: The system is still quite sensitive to the parameter values for things like filter thresholds and throws out more false positives or misses cells altogether if these are not tweaked manually. Possible solutions include some sort of machine learning approach on a larger dataset to create a set of robust parameter ranges, along with exploring techniques for filtering that are less dependent on hard coded parameters.

Apart from the two flaws above, there are several other avenues that could be pursued to improve the system in general. What follows is a list of potential changes or improvements:

  • Consider histogram equalization [9], which is a technique for equalizing the lighting across frames and can be used to correct the bleaching of the images as time increases. Applying this technique before the current preprocessing module could yield some improvements in noise reduction and clarity.
  • Explore more complex thresholds. At the moment, the images are binarized using a naive threshold with a parameter of 25. Anything pixels above this value are pushed to max, while anything below is pushed to zero. There are considerably more advanced thresholding techniques available that can provide a less parameter dependent result. An example of a more advanced algorithm would be Otsu's method for adaptive thresholding [10]
  • Decision Tree classification of cells based on morphological characteristics. This technique has been explored in [7] and could be extended to the current pipeline. This would be one way of creating robust parameters for filtering that could go towards solving the manual tuning problem.
  • Related to the above, filtering of blobs is very naive, looking only at basic morphological charactersitics like volume and area. More complex shape features such as roundness, presence of protrusions or convexity could provide more distinguishing power and really leverage the information the system has available to it.

Analysis

The analyses on Aspect Ratio and Polarity showed no correlations of teh type I was expecting; Aspect ratio did not correlate with speed and Polarity was not correlated to the direction as cell was moving in. This is interesting, as both of these were though to be fairly basic assumptions. These analyses are useful given our lack of knowledge about this niche and also feed back into the segmentation aspect of the project, as understanding how the cell behaves provides more criteria for filtering and segmentation. There are still many more aspects of HSC morphology that could be examined and a few are presented below.

A work in progress that was not complete for this report was the quantification of HSCs into categories - such as round, amoeboid or elongated - based on morphological measures like roundness, aspect ratio and circularity. This could also benefit from the decision tree classifier mentioned above to generate thresholds for the shape classification. The goal would be to understand if the cell shows preference for certain shapes based on its speed or environment.

Another avenue to explore is the presence of protrusions in the cell and the direction and length of these protrusions. This information would attempt to be correlated with characteristics like proximity to other cells or to niche components with the goal of understanding the causes behind this behaviour.
